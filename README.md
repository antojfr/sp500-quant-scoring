# Quantitative Stock Scoring on the S&P 500 with Machine Learning

End-to-end pipeline to **select S&P 500 stocks each month** that are likely to outperform the market, based on financial features, then **backtest** the resulting strategy — with a rigorous, look-ahead-free methodology.

> **Educational goal**: demonstrate mastery of the full quantitative strategy lifecycle — feature engineering, temporal validation, modeling, and *honest* risk-adjusted performance evaluation.

---

## Results at a glance

The **long-only (top 20%)** strategy outperforms the S&P 500 in absolute return, but rigorous analysis shows that **this outperformance comes essentially from market exposure (beta), not from a statistically significant selection alpha**.

| Metric (out-of-sample 2019–2024) | Top 20% | S&P 500 |
|---|---|---|
| Annualized return (net of costs) | **26.0%** | 15.8% |
| Sharpe ratio (net) | **0.93** | 0.85 |
| Max Drawdown | −29.5% | ~−24% |
| Beta (vs S&P 500) | 1.33 | 1.00 |
| Annualized alpha | +4.8% | — |
| **Alpha t-stat** | **1.12 → not significant** | — |

**Assumed conclusion**: on quasi-efficient markets, a weak ML signal (AUC ≈ 0.52) generates return, but no robust alpha once market risk is controlled for. This result — obtained without methodological bias — is the core of the exercise.

## What this project demonstrates

- **Financial feature engineering**: 24 features (momentum, risk, technical, macro, fundamentals) computed **by hand**.
- **Temporal rigor**: no information leakage (walk-forward validation, *cross-sectional* normalization, *train-only* standardization).
- **From-scratch implementation**: regularized (L2) logistic regression coded by hand (vectorized gradient descent), **validated** by matching `scikit-learn` to the 4th decimal.
- **Professional backtest**: Sharpe, alpha/beta (CAPM), max drawdown, turnover, transaction costs, market-neutral long-short variant.
- **Intellectual honesty**: beta/alpha decomposition, significance testing, documented biases.

## Methodology

### 1. Data
- **Universe**: S&P 500 constituents (~440 valid stocks / month after cleaning).
- **Period**: 2014–2024 (usable data: **Oct. 2015 → Dec. 2024**).
- **Frequency**: monthly (end-of-month rebalancing).
- **Sources**: `yfinance` (adjusted prices, volumes, dividends, shares outstanding), VIX, US rates.
- **Target**: binary classification — does the stock outperform the market **median** next month?

### 2. The 24 features (computed manually)
| Family | Features |
|---|---|
| Momentum (8) | 1/3/6/12-month returns, absolute and relative to the S&P 500 |
| Risk (5) | 1/3/6-month volatilities, 12-month beta, 3-month max drawdown |
| Technical (5) | RSI 14d, MACD, price/MA50, price/MA200, volume ratio |
| Macro (4) | VIX, yield curve, 10-year rate, S&P 500 monthly return |
| Fundamentals (2) | dividend yield, market capitalization (log) |

### 3. Preprocessing (look-ahead-free)
- **Winsorization** of outliers (1st / 99th percentile), *cross-sectionally* per date.
- **Log** of market cap (log-normal distribution → *Size* factor).
- **Cross-sectional z-score normalization** per date (consistent with the cross-sectional target, robust to market regimes).
- Filtering of low-count months (< 100 stocks) for a meaningful cross-section.

### 4. Modeling — Walk-Forward Validation
Temporal validation with an expanding window (no classic K-Fold, which would leak the future):
```
Train 2015→2018 → Test 2019
Train 2015→2019 → Test 2020
...           → ...
Train 2015→2023 → Test 2024
```

| Model | Mean AUC (6 folds) |
|---|---|
| **Logistic Regression** (baseline, from scratch) | **0.516** |
| MLP (32, 16) | 0.512 |
| Random Forest | 0.506 |

> Takeaway: on data with a low signal-to-noise ratio, the **linear** model beats more complex non-linear models, which mostly add variance. The exploitable cross-sectional signal is essentially **linear**.

### 5. Backtest
- Portfolio = **top 20%** of scores each month, equal-weighted.
- Compared against **two** benchmarks: the S&P 500 (cap-weighted) **and** the equal-weight universe (to isolate pure stock-picking from the equal-weight bias).
- **Long-short** variant (long top 20% / short bottom 20%): residual beta of 0.53, non-significant alpha (t = 0.58).

## Biases and limitations (acknowledged)

| Bias / limitation | Status |
|---|---|
| **Look-ahead bias** | Avoided (walk-forward, cross-sectional normalization, train-only fit) |
| **Survivorship bias** | Present — current S&P 500 composition used |
| **`market_cap` availability** | Reliable history from Oct. 2015 (`get_shares_full`) → dataset truncated upstream |
| **Transaction costs** | Simply modeled (10 bps/trade); slippage and market impact not modeled |
| **Non-significant alpha** | Honest result — no robust outperformance after beta control |

## Possible improvements
